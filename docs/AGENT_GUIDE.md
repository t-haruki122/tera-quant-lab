# Stock Data API - 開発者・エージェント向けガイド

このドキュメントは、本プロジェクトの設計方針、アーキテクチャ、および主要な実装詳細をAIエージェントや開発者が素早く理解し、効率的に開発・メンテナンスを行えるようにするために作成されました。

---

## 🚀 プロジェクト概要

株価データの取得、財務指標の分析、ニュースの集約、およびウォッチリスト管理を提供するフルスタックのWebアプリケーションです。

- **Backend**: FastAPI (Python) + SQLAlchemy (Async)
- **Frontend**: Vanilla JavaScript (SPA) + Vanilla CSS + Chart.js
- **Database**: SQLite
- **Data Source**: yfinance (株価・指標), GNews (ニュース)

---

## 🏗️ アーキテクチャと設計方針

### 1. バックエンド設計 (app/)
ディレクトリ構造は関心の分離に基づいています。

- **`routers/`**: APIエンドポイントの定義。リクエストの受け付けとバリデーション。
- **`services/`**: ビジネスロジック。外部API（yfinance等）との通信や指標計算を担当。
- **`models/`**: SQLAlchemyを用いたデータベースモデル（テーブル定義）。
- **`schemas/`**: Pydanticを用いたデータバリデーションとシリアライズ用モデル。
- **`clients/`**: 外部APIクライアント。サードパーティ製ライブラリのラッパー。

#### 💡 設計の重要ポイント
- **非同期処理**: `AsyncSession` を用いて、DBアクセスと外部API通信を非同期（`async/await`）で行い、スケーラビリティを確保しています。
- **キャッシュ戦略**: 外部APIへの負荷低減とレスポンス高速化のため、SQLiteをキャッシュとして利用しています（詳細は `stock_service.py` を参照）。
- **統計ミドルウェア**: `stats.py` を通じて、APIの呼び出し回数、キャッシュヒット率、レスポンスタイムをリアルタイムで記録しています。

### 2. フロントエンド設計 (frontend/)
シンプルさとパフォーマンスを重視し、ビルドステップのないバニラな技術スタックを採用しています。

- **Single Page Application (SPA)**: `index.html` の一部を JavaScript で動的に書き換えることで遷移を行います。
- **ES Modules構成**: `index.html` は `type="module"` で `app.js` のみを読み込み、機能別モジュールを `import/export` で連携します。
- **モダンなCSS**: 変数（`--accent-blue`など）を活用し、ダークテーマと洗練されたUIを実現しています。
- **状態管理**: `js/state.js` の `state` オブジェクトにUI状態を集約し、各モジュールで共有します。

#### フロントエンド主要モジュール

- `app.js`: エントリポイント。`App` クラスで初期化し、依存注入可能な構成。
- `js/state.js`: 共有状態と定数。
- `js/api.js`: `fetchAPI` とレスポンスキャッシュ。
- `js/utils.js`: 表示フォーマット、トースト、エスケープ処理。
- `js/stats.js`: `/stats` の取得とダッシュボード描画。
- `js/detail.js`: 詳細画面の検索・プロフィール・ニュース・財務データ読み込み。
- `js/chart.js`: 価格/RSI/SMAチャートと期間切替。
- `js/financial-chart.js`: 財務推移・配当推移チャートの描画。
- `js/list.js`: ウォッチリスト、タグ、フィルタ、並べ替え。
- `js/auth.js`: 認証モーダルとアカウントメニュー。

#### 互換レイヤー

HTMLの inline handler（`onclick` / `onchange`）は撤去済みです。イベントは各モジュールの `bind...Events` で登録されます。

## 📊 主要機能の実装詳細

### 株価と技術指標の計算
バックエンドの `stock_service.py` 内で、SMA（単純移動平均）やRSI（相対力指数）を計算しています。これにより、フロントエンドでは計算を行わず、提供されたデータを Chart.js で描画するだけに留めています。

### 通貨換算
`forex_service.py` は為替レートを取得し、フロントエンドではこれを用いて、米国株などの価格をユーザーの好みに応じて日本円等に換算して表示します。

### ユーザー・ウォッチリスト
ユーザーはアカウントを作成し、特定の銘柄を「ウォッチリスト」に保存できます。各銘柄には任意の「タグ（例：高配当、テック）」を付与でき、タグによるフィルタリングが可能です。

## 🛠️ 開発・拡張ガイドライン

### 新規APIの追加手順
1. `app/schemas/` にリクエスト/レスポンス用のPydanticモデルを作成。
2. `app/models/db_models.py` に必要に応じてDBテーブルを定義し、マイグレーション（または `init_db`）を実行。
3. `app/services/` にビジネスロジックを実装。
4. `app/routers/` にエンドポイントを追加。
5. `app/main.py` でルーターを `include_router` する。

### フロントエンドの修正手順
- **スタイル**: `index.css` の CSS 変数を使用してデザインの一貫性を保ってください。
- **API通信**: `js/api.js` の `fetchAPI` を利用してください。
- **状態更新**: 直接グローバル変数を作らず、`js/state.js` の `state` を更新してください。
- **機能分割**: 詳細画面の財務グラフロジックは `js/financial-chart.js` に配置してください。
- **イベント登録**: HTML属性でのイベント定義は追加せず、モジュール内で `addEventListener` を使用してください。

## 📝 統計とモニタリング
`/stats` エンドポイントで取得できるデータは、アプリケーションの健全性を維持するために重要です。キャッシュヒット率が著しく低い場合や、エラー率が高い場合は、キャッシュ TTL や外部 API の制限を確認してください。
